### **Common CodeQL Findings (and fixes):**

**1. Hardcoded Secrets:**
```csharp
// ? BAD
string apiKey = "12345-secret-key";

// ? GOOD
string apiKey = _configuration["ApiKey"];
```

**2. SQL Injection (if found):**
```csharp
// ? BAD
var query = $"SELECT * FROM Users WHERE Id = {userId}";

// ? GOOD (you already do this!)
var user = await _context.Users.FindAsync(userId);
```

## ?? **GITHUB ACTIONS WORKFLOW (Advanced)**

### **Create CI/CD Pipeline**

Create `.github/workflows/build-and-test.yml`:

```yaml
name: .NET Build and Test

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup .NET
      uses: actions/setup-dotnet@v3
  with:
        dotnet-version: 8.0.x
    
    - name: Restore dependencies
   run: dotnet restore
    
    - name: Build
      run: dotnet build --no-restore --configuration Release
    
    - name: Test
      run: dotnet test --no-build --verbosity normal --configuration Release
```

Commit and push:
```bash
mkdir -p .github/workflows
# Create the file above
git add .github/workflows/build-and-test.yml
git commit -m "Add CI/CD pipeline"
git push
```

## ?? **TROUBLESHOOTING**

### **Issue: Git push fails with authentication error**
```
Solution:
1. Use Personal Access Token instead of password
2. GitHub ? Settings ? Developer settings ? Personal access tokens
3. Generate new token with 'repo' scope
4. Use token as password when pushing
```

### **Issue: "Permission denied" when pushing**
```bash
Solution:
# Check remote URL
git remote -v

# If using HTTPS, ensure you have access
# If using SSH, ensure SSH key is added to GitHub
```

### **Issue: Too many files being pushed**
```bash
Solution:
# Check .gitignore is working
git status

# Remove files that shouldn't be tracked
git rm --cached <file>
git commit -m "Remove unnecessary files"
```

### **Issue: CodeQL scan fails**
```
Solution:
1. Check Actions tab for error details
2. Ensure .NET 8 is properly configured
3. Re-run the scan
```
